# Contributing Guidelines

Thank you for considering contributing to this Microsoft Office Add-in
project! Please follow these guidelines to ensure a smooth process.

## Pull Requests

- Fork the repository and create your branch from `main`.
- Write clear, descriptive commit messages.
- Ensure your PR addresses a single issue or feature.
- Reference related issues in your PR description.
- Run all tests and verify the add-in loads in Office before submitting.

## Code Style

- Use consistent indentation (2 spaces).
- Follow [Airbnb JavaScript Style Guide](https://github.com/airbnb/javascript).
- Include JSDoc comments for all public functions and classes.
- Use descriptive variable and function names.
- Prefer ES6+ syntax.

## Other Provisions

- All contributions must be your own work.
- By submitting a PR, you agree to license your contribution under the project's 
  license.
- Be respectful and constructive in all communications.

Thank you for helping improve this Office Add-in!
